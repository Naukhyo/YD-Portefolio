Hébergement d'un site avec GitHub Pages

Mise en ligne d'un site statique sur GitHub Pages via GitHub Desktop — sans serveur, sans abonnement, accessible 24h/24.

GitHub GitHub Desktop Hébergement Site statique
← Retour aux procédures

Présentation

GitHub Pages permet d'héberger gratuitement un site statique (HTML/CSS/JS) directement depuis un repo GitHub public. Le site est hébergé sur les serveurs de GitHub — le PC peut être éteint, le site reste en ligne.

GitHub Pages est gratuit pour les repos publics. L'URL générée est du type pseudo.github.io/nom-du-repo.

1. Créer un compte et un repo

Se connecter sur github.com, cliquer sur le + en haut à droite puis New repository.

Menu GitHub — New repository

Remplir le nom du repo, sélectionner Public et cliquer Create repository.

Création du repo — visibilité Public obligatoire pour GitHub Pages gratuit

Le repo apparaît dans My repositories

2. Pourquoi GitHub Desktop

L'interface web de GitHub limite l'upload à 100 fichiers max. Pour un site avec beaucoup d'images, il faut passer par GitHub Desktop qui n'a pas de limite.

Repo vide après création — on utilise GitHub Desktop pour uploader

L'interface web bloque à 100 fichiers — GitHub Desktop contourne cette limite

3. Installer et configurer GitHub Desktop

Télécharger GitHub Desktop sur desktop.github.com, l'installer et se connecter avec le compte GitHub.

GitHub Desktop — écran d'accueil après connexion

Cliquer Clone a repository from the Internet, sélectionner le repo et choisir le dossier local où il sera cloné.

Sélection du repo à cloner et du chemin local

4. Uploader les fichiers

Une fois le repo cloné, cliquer Show in Explorer pour ouvrir le dossier local. Copier-coller tous les fichiers du site dedans (HTML, CSS, images).

GitHub Desktop détecte automatiquement les nouveaux fichiers. Renseigner un message dans Summary et cliquer Commit to main.

178 fichiers détectés — commit avant envoi

Cliquer ensuite Publish branch pour envoyer les fichiers sur GitHub.

Publish branch — les fichiers sont envoyés sur les serveurs GitHub

Push en cours — progression visible en haut de GitHub Desktop

5. Activer GitHub Pages

Sur github.com, aller dans le repo → Settings → Pages. Sous Branch, sélectionner main et /(root) puis cliquer Save.

Configuration GitHub Pages — branch main, dossier root

Le site est en ligne en 2-3 minutes à l'adresse pseudo.github.io/nom-du-repo. Pour mettre à jour le site, il suffit de modifier les fichiers dans le dossier local, faire un nouveau commit et push depuis GitHub Desktop.

Le site reste en ligne même si le PC est éteint. GitHub Pages est hébergé sur les serveurs de GitHub, pas en local.
