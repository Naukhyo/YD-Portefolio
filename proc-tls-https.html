TLS & HTTPS

Fonctionnement du protocole TLS, différences entre HTTP et HTTPS, et lecture d'un certificat dans le navigateur.

← Retour aux procédures

HTTP vs HTTPS

HTTP (HyperText Transfer Protocol) est le protocole de base du web. Le problème : les données transitent en clair. N'importe qui sur le réseau peut les intercepter et les lire. HTTPS, c'est HTTP + TLS. Le trafic est chiffré de bout en bout entre le navigateur et le serveur.

Le cadenas dans la barre d'adresse indique une connexion HTTPS sécurisée

Comment fonctionne TLS

TLS (Transport Layer Security) assure trois choses : l'authentification du serveur (via son certificat), le chiffrement des données (via une clé de session négociée), et l'intégrité (les données ne peuvent pas être modifiées sans que ça se voie). La négociation se fait lors d'un "handshake" au début de la connexion.

Schéma TLS — échanges entre client, serveur et autorité de certification
TLS 1.2 et TLS 1.3 sont les versions actuelles. TLS 1.0 et 1.1 sont obsolètes et désactivés dans les navigateurs modernes. SSL est encore plus vieux et ne doit plus être utilisé.

Lire un certificat dans le navigateur

En cliquant sur le cadenas dans la barre d'adresse, on peut consulter le certificat du site : qui l'a émis (l'autorité de certification), pour quel domaine il est valable, et sa date d'expiration. Sur cyber.gouv.fr par exemple, le certificat est émis par DHIMYOTIS.

Un site avec un certificat expiré ou non valide affiche un avertissement de sécurité. Dans ce cas, ne jamais saisir de données personnelles ou d'identifiants.
