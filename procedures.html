Réseau — Simulation

Cisco Packet Tracer — Adressage IP et routage

Packet Tracer c'est le simulateur réseau de Cisco. On l'utilise pour monter des topologies réseau, configurer des routeurs et des switches en CLI, et tester la connectivité entre machines — sans avoir de vrai matériel.

Dans ce TP, j'ai mis en place un réseau avec plusieurs sous-réseaux, configuré le routage statique et vérifié la connectivité de bout en bout avec des ping.

Vue d'ensemble de la topologie dans Packet Tracer

Topologie avec plusieurs sous-réseaux interconnectés

Configuration d'une interface routeur

La configuration d'une interface se fait en mode privilégié puis en mode de configuration d'interface. Exemple pour affecter une IP sur fa0/0 :

# Accès en mode config
Router> enable
Router# configure terminal

# Config de l'interface
Router(config)# interface fa0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# exit

Configuration d'une interface en CLI

Ajout d'une route statique vers un autre réseau

Route statique et vérification

Pour que deux sous-réseaux puissent communiquer via un routeur, il faut soit du routage dynamique, soit des routes statiques. Dans ce TP on a utilisé des routes statiques — c'est plus simple à comprendre pour commencer.

# Ajouter une route statique
Router(config)# ip route 192.168.2.0 255.255.255.0 192.168.1.2

# Vérifier la table de routage
Router# show ip route

Test de connectivité entre deux sous-réseaux après configuration du routage

La commande show ip route affiche la table de routage. Les routes en S sont statiques, celles en C sont directement connectées. C'est utile pour déboguer quand un ping ne passe pas.
Réseau — Switching

VLANs sur switch Cisco

Les VLANs (Virtual LAN) permettent de segmenter un réseau physique en plusieurs réseaux logiques isolés. Même branché sur le même switch, un poste en VLAN 10 ne peut pas communiquer directement avec un poste en VLAN 20.

Dans ce TP j'ai configuré 3 switches avec 2 VLANs : VLAN 10 pour les postes PC1 à PC3, et VLAN 20 pour les postes PC3 à PC5. Les liens entre switches sont configurés en mode trunk pour laisser passer les deux VLANs.

Topologie complète — 3 switches interconnectés avec 2 VLANs

Plan d'adressage — VLAN 10 (192.168.10.x) et VLAN 20 (192.168.20.x)

Création des VLANs et affectation des ports

# Créer les VLANs
Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name VLAN-Users
Switch(config-vlan)# exit
Switch(config)# vlan 20
Switch(config-vlan)# name VLAN-Servers
Switch(config-vlan)# exit

# Affecter un port en mode access (VLAN 10)
Switch(config)# interface fa0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
Switch(config-if)# exit

# Configurer un port trunk entre switches
Switch(config)# interface fa0/24
Switch(config-if)# switchport mode trunk

Configuration de l'interface VLAN 10 avec l'IP de management

Vérification de l'isolation

Après la configuration, un ping entre deux postes du même VLAN doit fonctionner. Un ping vers un poste d'un autre VLAN doit échouer — c'est ce qu'on attend. Pour que deux VLANs puissent communiquer, il faut un routeur ou un switch L3.

Ping réussi entre deux postes du même VLAN — isolation vérifiée

Le mode trunk sur les liens inter-switches est important : sans ça, seul le VLAN natif passe et les autres VLANs sont bloqués. La commande show vlan brief permet de vérifier quels ports appartiennent à quel VLAN.
Réseau — Firewall

pfSense — VLANs et règles de filtrage

pfSense est un pare-feu open-source basé sur FreeBSD. Dans mon infrastructure (Laniakea), il fait le lien entre le WAN (réseau de l'école) et le LAN interne (192.168.10.0/24). Il gère aussi les VLANs pour séparer le réseau filaire, le WiFi d'entreprise et le WiFi invité.

La configuration des VLANs dans pfSense passe par l'interface web. On crée les VLANs sur l'interface LAN, on leur assigne une interface pfSense, et on configure les règles de filtrage pour définir ce qui peut passer ou non.

Liste des interfaces VLAN configurées dans pfSense

Créer un VLAN dans pfSense

Depuis l'interface web, aller dans Interfaces → Assignments → VLANs. Renseigner l'interface parente (le LAN physique), le tag VLAN et une description. Une fois créé, il faut assigner le VLAN à une interface pfSense et lui donner une IP (ex. 192.168.20.1/24).

Création d'un VLAN — tag 10, interface LAN, description "LAN Natif"

Règles de filtrage

Chaque interface VLAN a ses propres règles de pare-feu. Par défaut tout est bloqué — il faut créer des règles explicites pour autoriser le trafic. J'ai créé une règle pour autoriser le VLAN WiFi Corporate à accéder à internet, tout en bloquant l'accès aux autres VLANs.

Règle de filtrage — autorisation du VLAN WiFi Corporate vers internet

L'ordre des règles dans pfSense est important : elles sont évaluées de haut en bas et la première règle qui correspond est appliquée. Une règle "bloquer tout" en bas sert de filet de sécurité.
Administration — Helpdesk

GLPI — Installation et gestion des tickets

GLPI (Gestionnaire Libre de Parc Informatique) est un outil open-source pour gérer le parc informatique et les demandes de support. Il permet de centraliser les tickets d'incident, l'inventaire des équipements, et les contrats.

Dans mon infrastructure, GLPI est déployé sur la VM Titan (192.168.10.6). L'installation se fait via un assistant web qui configure la connexion à la base de données MySQL et crée les tables.

Installation

Après avoir déposé les fichiers GLPI dans le répertoire web du serveur, l'assistant d'installation guide la configuration de la BDD. Il faut renseigner le host MySQL, le nom de la base, et les identifiants.

Étape 1 de l'installation — configuration de la connexion à la base MySQL

Page de connexion GLPI après installation

Tableau de bord et tickets

Une fois connecté, le tableau de bord donne une vue d'ensemble des tickets en cours, des équipements, et des statistiques. Un technicien peut créer, prendre en charge et résoudre des tickets directement depuis l'interface.

Tableau de bord GLPI — vue d'ensemble des tickets et du parc

Après l'installation, il faut supprimer le dossier install/ pour des raisons de sécurité. GLPI affiche un avertissement si ce dossier est encore présent.
Sauvegarde — Backup

Veeam Agent — Sauvegarde et restauration

Veeam Agent for Windows est un outil de sauvegarde qui permet de sauvegarder un poste Windows sur un disque local, un partage réseau, ou un dépôt Veeam. Il supporte la sauvegarde complète, incrémentale, et la restauration fichier par fichier ou restauration complète du volume.

Dans ce TP, j'ai configuré un job de sauvegarde en mode "Entire computer" vers un partage réseau, puis j'ai testé la restauration.

Créer un job de sauvegarde

Étape 1 — Nommer le job de sauvegarde

Étape 2 — Mode "Entire computer" (sauvegarde complète)

Le mode "Entire computer" sauvegarde tout le système, y compris l'OS et les données. C'est le plus pratique pour une restauration bare metal (remplacer un disque dur en panne par exemple). Le mode "Volume level backup" permet de choisir des volumes spécifiques.

Étape 3 — Destination vers un partage réseau

Étape 4 — Planification (quotidienne, hebdomadaire...)

Récapitulatif du job créé — mode, destination, planification

Restauration d'un volume

La restauration se lance depuis Veeam Agent → clic droit sur le job → Restore. On sélectionne le point de restauration (date/heure), puis le mode de restauration : volume entier, fichiers individuels, ou restauration bare metal depuis le media de récupération.

Sélection du backup source pour la restauration de volume

Veeam crée aussi un media de récupération (ISO ou clé USB) qui permet de démarrer un PC hors service et de le restaurer directement — même si l'OS est inaccessible. C'est indispensable en cas de panne disque.
Système — Linux

Linux / Debian — Commandes de base et gestion de fichiers

Ce TP couvre la prise en main de Debian en ligne de commande : navigation dans l'arborescence, gestion des fichiers et dossiers, droits d'accès, et manipulation de texte. L'environnement utilisé est une VM Debian dans VirtualBox.

Navigation et gestion de fichiers

Les commandes de base pour se déplacer et gérer les fichiers sous Linux :

# Afficher le contenu d'un dossier avec les droits
yvan@vmdebian:~$ ls -la

# Créer un dossier et un fichier
yvan@vmdebian:~$ mkdir documents
yvan@vmdebian:~$ touch documents/essai.txt

# Afficher les infos d'un fichier (droits, taille, date)
yvan@vmdebian:~$ ls -l essai

ls -la — affichage des droits et propriétaires

ls -l — droits du fichier essai, propriétaire yvan

Manipulation de texte

Linux intègre des outils puissants pour traiter des fichiers texte directement en ligne de commande. grep permet de rechercher une chaîne dans un fichier, cat d'afficher son contenu, et diff de comparer deux fichiers.

cat et création de fichiers en terminal Debian

grep — recherche d'une chaîne dans un fichier

# Rechercher une chaîne dans un fichier
yvan@vmdebian:~$ grep "sun" res
here comes the sun
Following the sun

# Comparer deux fichiers
yvan@vmdebian:~$ diff etudiant yvanbis
1d0
< ceci est mon nom: Dupuit
Les droits sous Linux s'affichent en 3 groupes de 3 : rwx pour le propriétaire, le groupe, et les autres. chmod permet de les modifier. Par exemple chmod 644 fichier.txt donne lecture+écriture au propriétaire, lecture seule aux autres.
Réseau — Analyse

Wireshark — Analyse de trames réseau

Wireshark est un analyseur de protocoles réseau. Il capture tout le trafic qui passe sur une interface réseau et permet de l'analyser trame par trame. C'est un outil indispensable pour comprendre ce qui se passe sur un réseau et pour déboguer des problèmes de connexion.

Dans ce TP, j'ai capturé des échanges HTTP et analysé les trames pour identifier les couches du modèle OSI, les adresses IP et MAC, et les ports utilisés.

Interface Wireshark — liste des paquets capturés avec les colonnes Source, Destination, Protocol, Info

Analyser une requête HTTP

Une requête HTTP implique plusieurs échanges : la résolution DNS, l'établissement de la connexion TCP (handshake en 3 étapes), puis l'envoi de la requête GET et la réception de la réponse. Dans Wireshark, on peut filtrer par protocole pour isoler ce qu'on veut regarder.

Capture HTTP — échange TCP avec les détails du paquet déplié

Test de connectivité

En parallèle des captures Wireshark, j'ai utilisé des commandes ping pour générer du trafic ICMP et vérifier la connectivité. Les paquets ICMP apparaissent dans Wireshark et permettent de vérifier les temps de réponse.

Ping vers un serveur — les paquets ICMP sont visibles dans Wireshark

Filtres utiles dans Wireshark : http pour n'afficher que le trafic HTTP, tcp.port == 80 pour le port 80, ip.addr == 192.168.1.1 pour filtrer une IP. Le filtre tcp.flags.syn == 1 permet d'isoler les connexions TCP.
Sécurité — Chiffrement

Chiffrement avec GPG

GPG (GNU Privacy Guard) est un outil de chiffrement qui implémente le standard OpenPGP. Il permet de chiffrer des fichiers avec une clé publique/privée (chiffrement asymétrique) ou avec une passphrase (chiffrement symétrique).

Dans ce TP j'ai généré une paire de clés GPG, chiffré un fichier, et testé le déchiffrement. J'ai aussi exporté la clé publique pour la partager.

Principe du chiffrement asymétrique — clé publique pour chiffrer, clé privée pour déchiffrer

Générer une paire de clés

# Générer une paire de clés (interactive)
gpg --gen-key

# Ou version complète avec plus d'options
gpg --full-generate-key

# Lister les clés du trousseau
gpg --list-keys

Génération d'une clé GPG — saisie du nom et de l'email

Informations de la clé générée — RSA 3072 bits, empreinte unique

Chiffrer et déchiffrer un fichier

# Chiffrement symétrique (avec passphrase)
gpg --symmetric --armor TPgpg.txt

# Résultat : TPgpg.txt.gpg (fichier chiffré)

# Déchiffrer
gpg --decrypt TPgpg.txt.gpg

Fichier original + version chiffrée (.gpg) + export ASCII (.asc)

Pinentry — saisie de la passphrase pour déchiffrer

Le fichier .asc est une version en ASCII armor de la clé ou du message chiffré — utile pour partager par email ou copier-coller. L'option --armor produit ce format. Sans cette option, GPG génère un binaire .gpg.
Système — Windows Server

Windows Server — Ajout de rôles

L'ajout de rôles sous Windows Server se fait via le Gestionnaire de serveur. Un rôle c'est une fonctionnalité serveur : AD DS (Active Directory), DNS, DHCP, IIS... Chaque rôle a ses propres prérequis et son assistant de configuration post-installation.

Ce TP montre la procédure complète d'ajout de rôles, du lancement de l'assistant jusqu'à la vérification post-installation.

Lancer l'assistant

Depuis le Gestionnaire de serveur, aller dans Gérer → Ajouter des rôles et fonctionnalités. L'assistant s'ouvre et guide les différentes étapes.

Gestionnaire de serveur — accès au menu d'ajout de rôles

Assistant ajout de rôle — sélection d'AD DS

Configuration post-installation

Après l'installation d'un rôle, une notification apparaît dans le Gestionnaire de serveur pour lancer la configuration post-installation. Pour AD DS, ça consiste à promouvoir le serveur en contrôleur de domaine. Pour DHCP, c'est l'autorisation dans l'AD.

Configuration AD DS — assistant de promotion en contrôleur de domaine

Vérification prérequis avant installation — liste des checks

Avant d'ajouter des rôles, il vaut mieux faire les mises à jour Windows et configurer l'IP fixe du serveur. Si l'IP change après la promotion en DC, ça peut créer des problèmes avec le DNS et l'AD.
