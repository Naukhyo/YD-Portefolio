Protocole DHCP

TP DHCP sur Cisco Packet Tracer — configuration d'un serveur DHCP, pool d'adresses et attribution automatique aux clients.

← Retour aux procédures

Principe du DHCP

DHCP (Dynamic Host Configuration Protocol) permet d'attribuer automatiquement une configuration réseau à un client qui se connecte : adresse IP, masque de sous-réseau, passerelle par défaut, serveur DNS. Sans DHCP, il faudrait tout configurer manuellement sur chaque machine.

Le processus se fait en 4 messages : Discover (le client cherche un serveur), Offer (le serveur propose une IP), Request (le client accepte), Acknowledge (le serveur confirme). On abrège en DORA.

Dans mon infra, le DHCP est géré par pfSense (Laniakea) pour le réseau 192.168.10.0/24. Les serveurs ont des IP fixes (réservations statiques).

Topologie du TP

Le TP a été réalisé sur Cisco Packet Tracer. La topologie comprend un routeur qui fait aussi office de serveur DHCP, et plusieurs PC clients qui doivent obtenir leur configuration automatiquement.

1 Topologie Packet Tracer — routeur DHCP + clients

Configuration du pool DHCP

Sur le routeur Cisco, on configure le service DHCP en CLI. On crée un pool avec un nom, on définit le réseau distribué, la passerelle par défaut et le DNS. On peut aussi exclure certaines adresses du pool pour les équipements à IP fixe.

2 Création du pool DHCP en IOS
3 Réseau distribué et adresses exclues
Router(config)# ip dhcp excluded-address 192.168.1.1 192.168.1.10
Router(config)# ip dhcp pool MONRESEAU
Router(dhcp-config)# network 192.168.1.0 255.255.255.0
Router(dhcp-config)# default-router 192.168.1.1
Router(dhcp-config)# dns-server 8.8.8.8

Vérification — attribution aux clients

Une fois le DHCP configuré, les clients Packet Tracer obtiennent leur IP automatiquement. On vérifie avec ipconfig sur le client et show ip dhcp binding sur le routeur pour voir toutes les attributions actives.
